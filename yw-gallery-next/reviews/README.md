# Reviews

Codex records review findings here after Claude Code completes a feature slice.

## Naming

- Use the same slice name as the brief.
- Example: `M1-app-shell-review.md`.

## Review Focus

- Data loss or migration risk.
- Module responsibility boundaries.
- Mobile and touch usability.
- Duplicate event binding.
- Destructive actions without confirmation.
- Test and acceptance gaps.

Fix briefs must only ask Claude Code to address the listed review findings.
