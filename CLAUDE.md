# YW Gallery

纯前端单页应用，无框架，无构建工具。

## 技术栈
- index.html + app.js + styles.css，直接浏览器打开
- IndexedDB 持久化（localStorage 回退 + 自动迁移）
- 中文 UI，Apple 风格设计

## 数据模型
people → groups（大品类）→ categories（小品类）→ items（YW）

## 核心功能
- 四个主视图：主页体育生选择、体育生详情图库、添加 YW、设置/数据管理。
- 主页展示体育生卡片；体育生详情页按大品类、小品类展示 YW，并区分有图卡片与无图文本条目。
- 添加 YW 支持选择体育生、大品类、小品类，填写名称、数量、单位、日期，并标记“赠送”“现存”状态。
- 设置页支持体育生增删与图片更换；大品类/小品类增删、查看、折叠、拖拽排序；按体育生维度保存大小品类顺序。
- YW 支持新增、删除、状态切换、图片管理、同类型拖拽排序；图片支持上传、裁剪、多图轮播、添加、替换、删除。
- 数据持久化优先使用 IndexedDB，localStorage 作为回退并自动迁移；支持 JSON 导入导出备份。

## 三端兼容
- 全局原则：不改变数据结构、保存格式、业务流程和中文 Apple 风格。
- **Mac 与移动端是完全不同的交互逻辑**：Mac 依赖鼠标/键盘（hover、右键、HTML5 拖拽）；移动端依赖触摸（点按、滑动、长按），不共享 hover 和 HTML5 DnD。
- **Mac**：宽屏布局、顶部导航、多列表单、横向 YW 列表、鼠标拖拽排序。
- **iPad/iPhone（移动端）**：
  - iPad 横版：整体接近 Mac 布局，但触控目标加大、弹窗加宽、横向滚动适配触摸。
  - iPad 竖版：顶部导航允许换行；表单收敛为 1-2 列；禁止横向溢出。
  - iPhone：单列布局、控件全宽、弹窗底部弹出并适配安全区；卡片需进一步缩小以适配窄屏。
- 移动端交互原则：所有核心操作必须基于 click/touch 事件，不得依赖 hover、HTML5 拖拽等桌面专属能力。
